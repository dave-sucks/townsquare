We are building a web-first clone of Beli focused on: Google Places search, saving places, and viewing saved places on a map. Invites and streaks are OUT.

HARD UI RULES:
1) All non-map UI MUST use shadcn/ui components only (installed via shadcn CLI).
2) Do NOT write ad-hoc Tailwind class strings throughout pages.
   - If layout styling is needed, create minimal wrapper components using cva variants (PageShell, Stack, Container) and reuse them.
3) No other UI libraries. No MUI/Chakra/etc.

STACK:
- Next.js 14+ App Router + TypeScript
- shadcn/ui
- Prisma + Postgres
- Zod
- Auth: simple email/password (no invites)

GOOGLE INTEGRATION (REQUIRED):
- Use Google Places API to search and select places.
- Use the Google Place ID as the canonical unique identifier for places in our DB.
- Fetch place details to get: name, formatted address, lat/lng, types, price level (if available), and photo references (optional).

MAP (REQUIRED IN THIS MILESTONE):
- Use Google Maps JavaScript API to render a map on /map.
- Display pins for the logged-in user’s saved places using Place.lat/lng.
- Clicking a pin opens a shadcn Card overlay showing place name + address + status (Want/Been) and a button to open the place detail page.

MVP FEATURES TO BUILD NOW:
A) Project setup
   - Next.js + TS
   - Install shadcn properly (init + add components)
   - Add components: button, input, card, dialog, tabs, dropdown-menu, avatar, badge, textarea, toast/sonner, form, separator, skeleton

B) Prisma schema
   - User: id, email, passwordHash, username, createdAt
   - Place: id, googlePlaceId (unique), name, formattedAddress, lat, lng, primaryType?, types JSON?, priceLevel?, photoRefs JSON?, createdAt, updatedAt
   - SavedPlace: id, userId, placeId, status enum (WANT|BEEN), visitedAt?, createdAt, unique(userId, placeId)
   - (Optional) Review later

C) Pages
   - /signup, /login
   - /search: Google Places autocomplete search box. Selecting a place upserts Place and lets user save as WANT or BEEN.
   - /place/[id]: place details from DB + toggle save status
   - /map: Google map with pins for saved places (WANT/BEEN visually distinguishable if easy; if not, just pin all)

D) Architecture rules
   - Server actions or route handlers for mutations.
   - Zod validation for all incoming data.
   - Keep API keys in env vars.
   - Ensure saved places appear on the map immediately after saving (refresh data / revalidate).

WORKFLOW:
1) Install shadcn FIRST and render a basic page using it.
2) Then Prisma schema + migrate + seed (seed can be minimal since Google provides places).
3) Then implement auth.
4) Then implement Google Places search + save flow.
5) Then implement /map with pins.

Acceptance criteria:
- I can sign up and log in.
- I can search a real place using Google Places and save it.
- It appears pinned on /map at the correct location.
- Clicking the pin shows a shadcn overlay card and link to detail.

Start now by scaffolding the project and installing shadcn correctly before building the app pages.

MILESTONE CLEANUP v2 — MINDTRIP-STYLE APP SHELL + MODERN LAYOUT (NO NEW FEATURES)

Goal:
Refactor the UI to feel like Mindtrip (left sidebar, clean top controls, content+map split, detail drawer).
We keep all existing features and routes, but change layout and presentation.

HARD RULES:
1) Use shadcn/ui components only for UI (buttons, cards, tabs, sheets, dropdowns, inputs, command, etc).
2) Do NOT scatter Tailwind utilities across pages.
   - Only allow layout/styling inside a small set of wrapper components using cva variants.
   - Pages should mostly compose these wrappers + shadcn components.
3) No new UI libraries. Keep existing map integration as-is (Google map can remain a raw div/canvas).

TARGET IA (LEFT SIDEBAR LIKE MINDTRIP):
Primary navigation (left rail) with icons + labels:
- Map (dashboard)
- Home (feed)
- People
- Lists
- Saved (optional alias to Lists or a filtered view)
User avatar at bottom with dropdown.

LAYOUT REQUIREMENTS:
A) AppShell becomes a 2-column layout:
- Left: Sidebar (fixed width)
- Right: Main area
  - Top header row inside main: page title + optional actions
  - Then page body

B) Dashboard/Map page becomes a 2-pane split:
- Left pane: list + filters
- Right pane: map
Use shadcn ResizablePanelGroup so the user can resize.
The left pane must have its own ScrollArea.

C) Place details should open as a RIGHT DRAWER (Sheet) when coming from the dashboard.
- When user clicks a place in the list OR a map pin:
  - open a shadcn Sheet from the right
  - show PlaceDetails content inside
- Keep /places/[id] route for direct navigation, but on dashboard prefer drawer UX.

D) People + Profile pages should use the same shell and spacing.
- People page: search at top (Input), then list of user cards
- Profile page: profile header card + shadcn Tabs for Want / Been / Reviews / Lists

E) Feed page should feel like Mindtrip:
- A clean vertical feed
- Each feed item: Avatar + sentence + timestamp, then embedded place snippet card
- Reduce padding/whitespace: more “app-like,” less “giant docs page”

COMPONENT SYSTEM (CREATE THESE WRAPPERS, USE CVA VARIANTS):
1) AppShell
- renders Sidebar + main column

2) SidebarNav
- nav items list with icons (lucide)
- active route state
- bottom user menu (DropdownMenu)

3) PageHeader
- title + optional actions slot (right aligned)
- variant: default / dense

4) ContentContainer
- variant: default (max width), wide, full

5) SplitView
- wrapper around ResizablePanelGroup
- variants: mapSplit (list+map), contentSplit (content+details)

6) PlaceListRow
- compact modern row: name + secondary line + status pill + actions
- clicking selects place (for sheet)

7) PlaceDetailsPanel
- place name/address
- actions row: Want/Been toggle, Add to List, Open in Google Maps
- sections: Your Review, Friends who saved, Lists containing this place
- (No new features; just rearrange and modernize)

8) SearchBar (optional)
- shadcn Input or Command-based search
- keep it simple for now; no new backend needed

SHADCN COMPONENTS TO USE:
- Button, Card, Badge, Tabs, Sheet, ScrollArea, DropdownMenu, Avatar, Separator
- ResizablePanelGroup
- Optionally Command for search interactions

IMPLEMENTATION STEPS:
1) Build AppShell + SidebarNav and apply to all routes
2) Refactor Dashboard page into SplitView(mapSplit) with Resizable panels
3) Implement PlaceDetailsPanel inside a right-side Sheet on dashboard interactions
4) Refactor Home feed to use a consistent FeedItemCard pattern
5) Refactor People + Profile pages to match spacing and typography hierarchy
6) Ensure every page uses PageHeader and ContentContainer

ACCEPTANCE CRITERIA:
- Left sidebar exists and is consistent across pages
- Dashboard uses resizable split panes and looks like a modern map app
- Clicking a place on dashboard opens a right Sheet with details
- No giant empty whitespace / overly padded “dashboard cards” vibe
- Minimal page-level Tailwind; most styling in wrapper components via cva variants
- No database changes, no new product features
